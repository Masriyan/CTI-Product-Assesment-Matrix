# Contributing to CTI Product Assessment Matrix

Thank you for your interest in contributing to the CTI Product Assessment Matrix! This document provides guidelines for contributing to this project.

## How to Contribute

### Reporting Inaccuracies or Updates

If you notice outdated information or inaccuracies:

1. Open an issue describing the inaccuracy
2. Provide reliable sources for the correct information
3. Include the product name and section that needs updating

### Adding New Products

To suggest a new CTI product for inclusion:

1. Ensure the product is enterprise-grade or has significant market presence
2. Open an issue with the product name and tier classification
3. Provide publicly available information about the product
4. Include official sources (vendor website, analyst reports, etc.)

### Updating Existing Assessments

To update an existing product assessment:

1. Fork the repository
2. Create a new branch (`git checkout -b update-product-name`)
3. Make your changes following the template structure
4. Ensure all information is sourced and verifiable
5. Submit a pull request with a clear description of changes

## Assessment Template Structure

Each product assessment should follow this structure:

```markdown
# [Product Name]

## Overview
Brief description of the platform and vendor.

## Key Features
- Feature 1
- Feature 2
- Feature 3

## Pros
- Advantage 1
- Advantage 2
- Advantage 3

## Cons
- Limitation 1
- Limitation 2
- Limitation 3

## Use Cases
- Ideal use case 1
- Ideal use case 2

## Integration Capabilities
Description of API, SIEM integrations, and interoperability.

## Pricing Model
General pricing structure (if publicly available).

## Target Organization Size
Enterprise, Mid-Market, SMB, etc.

## Sources
1. [Source 1](URL)
2. [Source 2](URL)
3. [Source 3](URL)

## Last Updated
Month Year
```

## Quality Guidelines

### Objectivity
- Maintain neutral, factual tone
- Base assessments on verifiable information
- Avoid promotional language or bias
- Present both strengths and weaknesses fairly

### Sources
- All claims should be backed by reliable sources
- Prefer official vendor documentation, analyst reports, and reputable security publications
- Include direct links to sources
- Avoid relying solely on marketing materials

### Accuracy
- Verify information is current (within the last 12 months)
- Cross-reference multiple sources when possible
- Clearly mark information that may be outdated
- Update "Last Updated" field when making changes

### Completeness
- Fill in all sections of the template
- If information is not publicly available, state "Information not publicly available"
- Provide context for technical features
- Include relevant industry certifications or compliance features

## Writing Style

- Use clear, concise language
- Avoid jargon where possible; define technical terms when necessary
- Use bullet points for easy readability
- Keep assessments between 300-800 words
- Use active voice

## What NOT to Include

- Personal opinions without factual basis
- Unverified claims or rumors
- Sensitive pricing information (unless publicly disclosed)
- Comparison rankings or "best product" recommendations
- Proprietary or confidential information
- Marketing hype or exaggerated claims

## Code of Conduct

### Our Standards

- Be respectful and professional in all communications
- Focus on constructive feedback
- Welcome diverse perspectives and experiences
- Assume good faith in contributions

### Unacceptable Behavior

- Harassment or discriminatory language
- Personal attacks or insults
- Sharing private information without permission
- Promoting specific products for commercial gain
- Deliberate misinformation

## Review Process

1. **Submission**: Submit pull requests with clear descriptions
2. **Review**: Maintainers will review for accuracy, objectivity, and completeness
3. **Feedback**: Contributors may be asked to provide additional sources or clarification
4. **Approval**: Once approved, changes will be merged
5. **Recognition**: Contributors will be acknowledged in the project

## Vendor Participation

Vendors are welcome to:
- Suggest corrections to factual errors about their products
- Provide links to updated documentation or resources
- Notify us of major product changes or rebranding

Vendors should NOT:
- Write or substantially edit their own product assessments
- Remove legitimate criticism or limitations
- Add promotional content

Vendor-submitted updates will be clearly marked and independently verified.
